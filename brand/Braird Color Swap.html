The free color swap, on the real components

Same Surfc components, same code — only colors.css changes. Left is today's warm Surfc; right is the Braird drop-in (forest ink, shoot-green, cool paper). Everything aliased to a colour token recolours with zero edits. Watch the note card on the right for what a token can’t reach.

Surfc— warm, today
Braird— colors.css swap only

✓ Swapped for free

  • Buttons, badges, chips, inputs, toasts — all accent/ink/border tokens flip to green + forest instantly.
  • Stat cards & source rows — the amber initial tile and figures pick up shoot-green from --color-accent-*.
  • Idea chips — canon ideas go green; “your” ideas shift to clay so they never collide with the new green accent.
  • Text, surfaces, dividers — walnut → forest ink, parchment → cool paper, everywhere at once.

✕ Needs a design pass

  • The note card’s amber dot-grid “paper” + folded dog-ear — hard-coded in the component, so they stay warm (see right column). The signature motif must be re-drawn, not recoloured.
  • Warm-brown shadows (shadows.css) — still cast amber; want a neutral/green-tinted set.
  • Book-spine source row — the shelf-shadow + spine groove read “library”; Braird’s metaphor is growth, not stationery.
  • Wordmark, fonts & voice — separate from colour entirely.
The 40% the token can’t reach
Everything on this card is Braird-green except the paper. The dot-grid texture and the dog-ear corner are painted with literal amber values inside NoteCard.jsx — no token points at them. This is the index-card motif that carries Surfc’s warmth, and it’s exactly the piece a rebrand has to redesign rather than retheme.
